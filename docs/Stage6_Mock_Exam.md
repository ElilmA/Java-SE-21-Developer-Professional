## Stage 6 - 模拟考试与错题整理（逐题辅导）

目标：熟悉题型节奏；通过模拟题查缺补漏；正确率稳定 75%–80%+。  
文档记录：本文件 `Stage6_Mock_Exam.md`。代码演练（可选）：`src/main/java/com/example/cert21/stage6_mock_exam`

### 任务与辅导

1) 选择 3–5 套高质量模拟/真题  
- 做什么：按正式考试时间完成一套并计时记分。  
- 提示：可用手机计时；不要边查资料边做。  
- 参考记录格式：
```
套题1：得分 82/100，用时 80 分钟
错题：Q5（考点：Stream 终端操作）、Q12（考点：模块依赖）
```

2) 错题归纳  
- 做什么：每道错题写清楚考点、错误原因、正确结论。  
- 提示：把结论简短写进本文件，便于回顾。  
- 参考条目：
```
Q5：Stream 终端操作
- 错因：把 reduce 和 collect 混淆
- 正解：collect 用于生成集合/映射，reduce 聚合成一个值
```

3) 差距弥补  
- 做什么：针对错题考点，回到对应阶段代码/文档重练；再做新题验证。  
- 提示：直到模拟分数稳定 75%–80%+。

### 提交与检查
- 错题与总结写在本文件；做完一套模拟题就更新一次。  
- 需要代码实验的概念可写到 `stage6_mock_exam` 包下的小类里。  
- 阶段完成提交示例：`git commit -m "Stage6 完成第N套模拟题总结"`。
